● 新加的这个 Claude 引擎命名是分层级的：

  引擎 ID（代码层）
  - ChatEngineId 类型新增成员："claude-code-native"（kebab-case）—
  src/types.ts:153
  - 在 isClaudeFamilyEngine() 里和 "claude" 一起被归为 Claude 家族 —
  src/lib/chatEngineIds.ts:5
  - 能力常量：CLAUDE_CODE_NATIVE_CAPABILITIES —
  src/components/chat/engineCapabilities.ts:15

  底层实现名
  - 后端 crate 叫 claude-code-rust，vendored 在 vendor/claude-code-rust/
  - README/架构文档里引用作 claude-code-native

  i18n / Onboarding 键
  - setup.json 里的选项键用
  camelCase：claudeCodeNative（src/i18n/resources/{en,pt-BR,zh-CN}/setup.json）

  用户可见的展示名

  ┌───────────────────────┬──────────────────────┬─────────────────────┐
  │         场景          │          en          │        zh-CN        │
  ├───────────────────────┼──────────────────────┼─────────────────────┤
  │ Onboarding / 文档全称 │ Claude Code (Native) │ Claude Code（内置） │
  ├───────────────────────┼──────────────────────┼─────────────────────┤
  │ app.json 短标签       │ Native               │ 内置                │
  └───────────────────────┴──────────────────────┴─────────────────────┘

  简言之：代码里是 claude-code-native，键名是 claudeCodeNative，底层是
  claude-code-rust，UI 上中文叫“Claude Code（内置）”、英文叫“Claude Code
  (Native)”。它和原有的 "claude"（Claude Agent SDK sidecar）是两个不同的引擎。

     Context

  但有个实现细节值得提一下

  当前 claude_code_native.rs::send_message 用的是非流式调用：

  let chat_future = client.chat(history.clone(), None);  // 不是 chat_stream
  // ...
  let assistant_text = chat_response.choices.first()...message.content...;
  if !assistant_text.is_empty() {
      event_tx.send(EngineEvent::TextDelta { content: assistant_text.clone()
  }).await;
  }

  也就是说，无论回复多长，都是等 API 返回完整 JSON 后，一次性作为单个 TextDelta
  emit 出来——没有 token-by-token
  的流式增量。对短回复（"PONG"）没问题，但回复一长，用户在 Panes
  前端会看到「转圈→整段突然出现」的体验，而不是逐字打字机效果。

  ApiClient 其实已经实现了 chat_stream()
  方法（vendor/claude-code-rust/src/api/mod.rs:78），返回 SSE
  流。如果想要真正的流式体验，可以把 send_message 改造成：
  1. 调 client.chat_stream() 拿到 reqwest::Response
  2. 按 data: {...} 行解析 SSE
  3. 每个 choices[0].delta.content 来一段就 emit 一个 TextDelta
  4. 流结束后 emit TurnCompleted

  要我把这个流式化改造做了吗？这会显著改善 Panes 里 Native 引擎的实际使用体验。
