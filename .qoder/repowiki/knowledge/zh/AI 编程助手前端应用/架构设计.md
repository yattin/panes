- **技术栈与框架**：采用 React + TypeScript 构建 UI，Vite 作为构建工具，Tauri 提供原生能力桥接（通过 `src/lib/ipc.ts`）。状态管理使用 Zustand (`src/stores/`)，国际化使用 i18next (`src/i18n/`)。
- **分层架构**：
  - **入口层**：`src/main.tsx` 负责引导初始化（Locale、I18n）并挂载根组件。
  - **布局与视图层**：`src/App.tsx` 处理全局快捷键、原生菜单事件监听及顶层模态框；`src/components/layout/ThreeColumnLayout.tsx` 实现可拖拽调整的三栏布局（侧边栏、主内容区、Git 面板）。
  - **业务组件层**：按功能域划分为 `chat`（对话交互）、`editor`（代码编辑与差异对比）、`terminal`（终端会话）、`git`（版本控制操作）、`workspace`（工作区配置）等子目录。
  - **状态管理层**：`src/stores/` 目录下包含 `chatStore`、`gitStore`、`terminalStore` 等，负责管理各模块的本地状态并与后端同步。
  - **通信层**：`src/lib/ipc.ts` 封装了所有与 Tauri 后端的 `invoke` 调用和 `listen` 事件订阅，是前后端交互的唯一通道。
  - **工具层**：`src/workers/` 包含 Web Workers 用于高性能的 Diff 和 Markdown 解析。
