- 业务域内严格遵循三层划分：`domain` 存放不可变逻辑与类型，`application` 定义 Store 与 Gateway 抽象，`infrastructure` 负责具体 IPC 调用与存储实现。
- 所有与后端交互的逻辑必须通过 `*Gateway` 或 `*Repository` 接口注入，禁止在 UI 组件或 Store 中直接调用 `@tauri-apps/api`。
- UI 组件通过 `use*Store` Hooks 订阅状态，并调用 Store 暴露的方法触发副作用，保持组件无状态化。
- 流式事件（如 Chat 输出）在 `infrastructure` 层监听并通过 Gateway 转发至 Store，Store 内部使用批处理机制更新状态以优化渲染性能。
